Troubleshooting

Audio Issues

No sound output

  • Check that Audio Output is connected in your patch
  • Verify your audio interface is selected in Options > Audio Settings
  • Check that your interface isn't being used by another application
  • Make sure output isn't muted (check the Output Toggle node if present)

Audio crackling or dropouts

  • Increase buffer size in Audio Settings (try 256 or 512 samples)
  • Close other applications using audio
  • Disable WiFi and Bluetooth if not needed
  • Check your audio interface drivers are up to date

High latency

  • Use ASIO drivers on Windows for lowest latency
  • Lower the buffer size (but watch for crackling)
  • Reduce the number of plugins in your chain

Plugin Issues

Plugin not appearing after scan

  • Make sure you're scanning the correct folder
  • Only VST3 plugins are supported (not VST2)
  • Check if the plugin is 64-bit (32-bit plugins won't work)
  • Some plugins may be blacklisted - check Options > Plugin Management

Plugin crashes on load

  • The plugin will be automatically blacklisted
  • Try updating the plugin to the latest version
  • Check Help > Event Log for error details

Plugin window doesn't appear

  • Double-click the plugin node to open its editor
  • Check if the window opened off-screen (resize/move your main window)
  • Some plugins don't have a GUI - their parameters appear in the node

Performance Issues

High CPU usage

  • Some plugins are CPU-intensive - check individual plugin CPU in the node
  • Freeze or bounce tracks you're not actively editing
  • Consider using fewer instances of heavy plugins

Getting Help

If you're still having issues:

  • Check Help > Event Log for error messages
  • Visit the GitHub Issues page
  • Include your OS version, audio interface, and steps to reproduce
